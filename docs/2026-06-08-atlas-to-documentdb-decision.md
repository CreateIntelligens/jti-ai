# 資料庫付費升級評估：Atlas 付費層 vs AWS DocumentDB

- **日期**：2026-06-08 ｜ **狀態**：Draft（待主管決策）

---

## 摘要（TL;DR）

我們的需求是「**付費升級資料庫**」+「**多地部署**」+「**連線簡單、維運成本低**」。

針對此需求評估了兩個付費方向：**Atlas 付費層（M10 / Flex）** 與 **AWS DocumentDB**。

關鍵差異：兩者價格相近（約 $57/月起），但在「多地連線方式」「維運負擔」「是否引入單點依賴」上落差明顯——詳見下方比較。
DocumentDB 的主要適用情境為「公司政策要求資料留在 AWS」或「後端會收斂進 AWS 同一內網」。

---

## 一頁比較

| 項目 | MongoDB Atlas 付費層 | AWS DocumentDB |
|---|---|---|
| 月費（入門） | Flex ~$8–30；M10 ~$57（含三節點 HA） | t4g.medium ~$56 起（單節點，HA 需再 ×2） |
| 多地連線 | 設 IP 白名單即可直連（`mongodb+srv://`） | 須架跳板機 + SSH Tunnel 或 VPN |
| 多區域部署 | 原生支援，含跨雲（AWS/GCP/Azure） | 僅限 AWS 生態圈內 |
| 延遲 | 可選離台灣近的區域（東京/新加坡） | 目前在美西，延遲較高 |
| I/O 費用 | 已含在方案內，帳單可預測 | 另計，高頻讀寫可能偏高 |
| 維運負擔 | 升級層級即可，無額外元件 | 需顧跳板、金鑰、隧道容器 |
| 程式改動 | 幾乎不用 | 需加隧道容器、多地配金鑰 |

## DocumentDB 在「多地部署」下的主要考量

**關鍵點：在多地架構下，它可能讓「多地分散」退化為「假的高可用」。**

我們做多地部署是為了**避免單點故障**。但 DocumentDB 沒有公開 IP，所有外部節點都必須**繞同一台 AWS 同區跳板**才能連到 DB：

```
   台灣節點 ─┐
   某地節點 ─┼─→ 同一台跳板（AWS 同區主機）─→ DocumentDB
   本機節點 ─┘            ↑
                  全體共同依賴的單點 —— 它一掛，所有地方的後端同時連不到 DB
```

➡️ **辛苦做的多地分散，被一個跳板單點抵消了。** 這比單機部署還脆弱，與當初多地化的目的自相矛盾。
（且若拿現有業務主機如 hciot 24h 機兼當跳板，等於讓它一台扛全體 DB 咽喉，職責混淆、風險集中。）

其他實務負擔：

1. **每次部署都多一個 tunnel 容器**：每台機器都要多維護一個隧道服務，部署複雜度永久增加。
2. **每台都要產生 / 配置金鑰**：金鑰散佈到各機器，管理麻煩，且每多一台就多一份外洩風險。
3. **成本相當卻更貴**：生產級 HA 下 DocumentDB 常比 Atlas 貴 25–30%（單節點價要 ×2 才有高可用）。
4. **延遲無法消除**：DB 在美西，台灣節點每次查詢繞太平洋。

> DocumentDB 真正划算的場景是「後端也都跑在 AWS 同一個內網內」。我們是多地、不固定部署，吃不到它的優點，只承擔它的限制。

## 補充事實

- **目前用量還很低**：Atlas M0 免費層僅用 **10.7%（54.8MB / 512MB）**，短期不會滿，無急迫性，可從容決定。
- **可行性已驗證**：DocumentDB 也已建好、Atlas 資料（19,119 筆）已完整搬入並驗證無誤——所以「技術上做得到」不是問題，問題是「**值不值得**」。
- **Atlas 原資料未動**，無論決定為何都可隨時回退。

## 附錄：技術備忘（給執行者，主管可略過）

**若決定走 Atlas 付費層**：
- 後台 Network Access 設 IP 白名單（或 VPC Peering / PrivateLink），`MONGODB_URI` 升級層級即可，程式碼不動。

**若決定走 DocumentDB（多地隧道容器化）**：
- 架構：每台機器 `docker compose` 加一個 `db-tunnel` 容器（autossh 自動重連）→ 連跳板 `52.12.0.227` → DocumentDB。backend 改連 `db-tunnel:27017`。
- 連線雷區：
  - mongosh / mongorestore 帶 URI 時，`&` 會被 shell 截斷而退回 localhost → 用顯式 flag 或單引號包住 URI。
  - DocumentDB 的 db-tools 用 `--ssl` / `--sslCAFile`（非 `--tls`）。
  - 經隧道連 localhost 時，URI 需加 `tlsAllowInvalidHostnames=true`、`directConnection=true`，並移除 `replicaSet`。
- 已完成：DocumentDB cluster（us-west-2, 5.0.1）、Atlas→DocumentDB 遷移（19,119 筆，索引齊全）、Atlas dump 備份於跳板 `~/atlas_dump_20260608/`。
- 向量資料存於本地 LanceDB，不在 DB 遷移範圍。
