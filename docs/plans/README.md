# Docs Plans Guide

This directory now keeps documents that are still useful as current reference material:

- runbooks and migration guides
- current feature/reference docs
- current specs and implementation summaries
- active plans that are still being executed

Completed implementation plans and historical feature notes belong in [`../archive/plans/`](../archive/plans/).

- `MIGRATION_GUIDE.md`
- `SESSION_STORAGE_README.md`

- `PROMPT_MANAGEMENT.md`
- `USER_API_KEY.md`
- `color_quiz_spec.md`
- `color_quiz_implementation_plan.md`

Rule of thumb:

- if a document describes the current system or an operational workflow, keep it here
- if a document mainly records how a finished change was implemented, move it to the archive
